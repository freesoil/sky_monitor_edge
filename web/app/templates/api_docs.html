üåü API Overview

The Edge Monitor API provides RESTful endpoints for controlling and monitoring ESP32-based edge camera devices. All endpoints return JSON responses and use standard HTTP status codes.

Base URL

http://your-server:8000

Content Type

All POST requests should include the header:

Content-Type: application/json

üîç Device Discovery

POST Discover Edge Devices

/api/camera/discover
Scans the local network for ESP32 edge monitoring devices and establishes connections.

Response:

{ "success": true, "device_ip": "192.168.1.100:80", "devices": ["192.168.1.100:80", "192.168.1.101:80"] }

üìπ Camera Settings

POST Update Camera Setting

/api/camera/setting
Updates individual camera parameters such as frame size, quality, brightness, etc.

Parameters:

Parameter Type Description Example Values
setting string Camera parameter name framesize, quality, brightness, contrast, saturation
value integer New value for the parameter 0-12 (framesize), 4-63 (quality), -2 to 2 (others)

Request Example:

{ "setting": "framesize", "value": 6 }

üéÆ Device Commands

POST Send Device Command

/api/camera/command
Sends control commands to the edge device for recording management and system control.

Available Commands:

Command Description Effect
start Start recording Begins video capture according to configured intervals
stop Stop recording Halts all video capture operations
pause Pause/resume system Toggles system pause state
restart Restart device Performs a complete system restart
photo Capture single photo Takes one photo and saves to SD card

‚ö†Ô∏è Error Handling

The API uses standard HTTP status codes and returns error information in JSON format:

Status Codes

Code Meaning Description
200 OK Request successful
400 Bad Request Invalid request parameters
503 Service Unavailable No device connected or device unreachable
500 Internal Server Error Server-side error occurred

Error Response Format

{ "success": false, "error": "Error description", "detail": "Additional error details" }

üíª Code Examples

JavaScript/Fetch

// Discover devices fetch('/api/camera/discover', { method: 'POST', headers: { 'Content-Type': 'application/json' } }) .then(response => response.json()) .then(data => console.log(data)); // Get device status fetch('/api/camera/status') .then(response => response.json()) .then(data => console.log(data)); // Update camera setting fetch('/api/camera/setting', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ setting: 'quality', value: 10 }) }) .then(response => response.json()) .then(data => console.log(data));

Python/Requests

import requests # Discover devices response = requests.post('http://your-server:8000/api/camera/discover') print(response.json()) # Get device status response = requests.get('http://your-server:8000/api/camera/status') print(response.json()) # Send command response = requests.post('http://your-server:8000/api/camera/command', json={'command': 'start'}) print(response.json())
